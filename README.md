## Overview

This repository contains the full data engineering and analysis pipeline for a study of
undergraduate academic performance at Mountain Top University. The study applies OLS regression
and time-series analysis to a real institutional dataset, supplemented by statistically
synthesized variables where original records were unavailable.

The pipeline proceeds in four stages:

1. **Enrichment** — synthesize three missing independent variables and derive a lagged GPA predictor
2. **Verification** — run an automated check suite (52 checks) to confirm dataset integrity before modelling
3. **Analysis** — OLS regression with full diagnostics, and CGPA trajectory analysis across
   Levels 100–400, delivered as a notebook (primary submission) and an interactive Streamlit app
4. **Prediction (secondary)** — k-fold out-of-sample metrics (`predict.py`) and operational CGPA
   scoring for new rows (`score.py`), using the same OLS specification as the notebook and app

## Quick Start

### Requirements

Python 3.9 or higher.

```bash
pip install -r requirements.txt
```

Dependencies: `streamlit`, `pandas`, `numpy`, `scipy`, `scikit-learn`, `statsmodels`, `plotly`,
`notebook`, `ipykernel`.

### Run the full pipeline

```bash
# 1. Regenerate the enriched dataset
python enrich.py

# 2. Verify all checks pass (exit code must be 0)
python verify.py

# 3. (Optional) Out-of-sample prediction metrics — k-fold OLS on enriched data
python predict.py --seed 42
# Exploratory Level-400 target only (OQ-1 — discuss with supervisor before using in the paper)
python predict.py --seed 42 --also-cgpa400

# 4. (Optional) Score new rows — CSV in, CSV out (same four predictors as the OLS model)
#    Each row needs Attendance_Rate, Study_Hours_Per_Week, Course_Load, plus either
#    Previous_GPA OR (CGPA100, CGPA200, CGPA300). Extra columns are ignored with a warning.
python score.py --input examples/example_score_rows.csv --output scored_output.csv

# 5. Open the analysis notebook
jupyter notebook model.ipynb

# 6. Launch the Streamlit presentation app
streamlit run app.py
```

> Do not proceed to modelling if `verify.py` exits with code 1.

**Scoring input contract:** The fitted model does **not** accept an arbitrary list of semester SGPA
values. It uses `Previous_GPA` (= mean of CGPA100–300 in the enrichment definition) plus the three
behavioural fields. Including `SGPA` as a predictor is an open research question (OQ-4 in
`context.md`); `score.py` ignores unused columns and prints which were ignored.

See `examples/example_score_rows.csv` for a minimal template.

## Dataset

### Source

**File:** `academic_performance_dataset_V2.csv`  
**Records:** 3,046 students across 17 programmes  
**Graduation years:** 2010–2014  

| Column | Description |
|---|---|
| ID No | Student identifier |
| Prog Code | Programme code (CIS, MAT, EEE, etc.) |
| Gender | Male / Female |
| YoG | Year of graduation |
| CGPA | Overall cumulative GPA (0–5 scale) |
| CGPA100–400 | Per-level GPA (Year 1 through Year 4) |
| SGPA | Semester GPA |

**Known data quality issue:** 70 ID numbers appear on rows with conflicting programme, gender,
or graduation year. These are data-entry errors in the source records — not real duplicates.
Both `verify.py` and `model.ipynb` flag or drop these rows before analysis.

### The modelling gap

Three of the four independent variables required by the regression model — `Attendance_Rate`,
`Study_Hours_Per_Week`, and `Course_Load` — are entirely absent from the source data.
Collecting them retrospectively was not feasible. They were synthesized statistically using
a defensible generative model. See [VARIABLE-SYNTHESIS.md](VARIABLE-SYNTHESIS.md) for the
full methodology.

### Enriched schema

After running `enrich.py` the dataset has 15 columns:

| Column | Source | Description |
|---|---|---|
| ID No | Original | Student identifier |
| Prog Code | Original | Programme code |
| Gender | Original | Male / Female |
| YoG | Original | Year of graduation |
| CGPA | Original | Overall cumulative GPA — **dependent variable** |
| Previous_GPA | **Derived** | `mean(CGPA100, CGPA200, CGPA300)` — lagged 3-year average |
| CGPA100–400 | Original | Per-level GPA |
| SGPA | Original | Semester GPA |
| Attendance_Rate | **Synthesized** | Class participation (%), range [40, 100] |
| Study_Hours_Per_Week | **Synthesized** | Weekly study hours, range [2, 20] |
| Course_Load | **Synthesized** | Courses registered per semester, range [8, 20] |
| Genotype | **Synthesized** | Haemoglobin genotype AA / AS / SS (multinomial, CGPA-independent) |

## Methodology Summary

### Regression model

```
CGPA ~ Previous_GPA + Attendance_Rate + Study_Hours_Per_Week + Course_Load + Genotype (dummies)
```

Fitted using OLS (`statsmodels`). All synthesis is seeded (default `--seed 42`) for full
reproducibility.

### Variable synthesis

Two scientific fixes distinguish the current version (v2) from the original approach (v1):

**Fix 1 — Endogeneity mitigation (blended synthesis)**

Behavioural variables use a blended generative model:

```
X = 0.60 × base  +  0.40 × correlated_component  +  noise
```

where `base` is drawn from a CGPA-independent population distribution. This reduces the
CGPA–predictor correlation from ~0.60 (v1) to ~0.25–0.28 (v2), reflecting a genuine but
weak behavioural relationship rather than a near-deterministic one.

**Fix 2 — Arithmetic overlap reduction**

`Previous_GPA = mean(CGPA100, CGPA200, CGPA300)` replaces the v1 definition of
`Previous_GPA = CGPA100`. The three-year average is a genuinely lagged quantity. CGPA400
(Level 400 performance) is the new information not yet captured in `Previous_GPA`.

See [VARIABLE-SYNTHESIS.md](VARIABLE-SYNTHESIS.md) for the full mathematical foundations,
programme-level parameters, and version history.

## Limitations

These must appear in any paper or report using this dataset.

**L1 — Residual endogeneity (most critical)**  
`Attendance_Rate` and `Study_Hours_Per_Week` are synthesized using CGPA as an input (40%
weight). The significant OLS p-values do not constitute evidence of causal relationships —
they confirm the regression machinery works on a dataset where the relationship was constructed.

**L2 — Arithmetic overlap in `Previous_GPA`**  
`Previous_GPA` shares three of the four components of `CGPA`. The Adj R² of 0.953 reflects
this definitional relationship, not purely empirical predictive power.

**L3 — Synthesized variables are not observed data**  
`Attendance_Rate`, `Study_Hours_Per_Week`, and `Course_Load` were never measured. Coefficient
estimates for these variables carry no empirical weight.

**L4 — Overload rate assumption**  
The 5% overload probability and 16–20 course ceiling are unverified assumptions calibrated
against a single MTU transcript.

**L5 — ID number collisions**  
70 ID numbers have conflicting attributes across rows — data-entry errors in the source.
Duplicate rows are dropped before analysis.

**L6 — Generalizability**  
Synthesis parameters are calibrated for a Nigerian university context and are not transferable
to other institutions.

**L7 — Synthesized genotype**  
`Genotype` was not collected from medical records. Values are drawn from fixed population
proportions (75\% / 24\% / 1\% for AA / AS / SS) independent of CGPA. Genotype coefficients
carry no clinical weight.

## Reproducibility

All synthesis is fully reproducible via the random seed:

```bash
python enrich.py --seed 42   # default
python enrich.py --seed 7 --output academic_performance_enriched_seed7.csv
python verify.py --input academic_performance_enriched_seed7.csv
```

Re-running with the same seed and input file always produces an identical enriched CSV.
